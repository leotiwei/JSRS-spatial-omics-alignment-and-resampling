# JSRS spatial omics alignment and resampling

This repository provides cleaned code and documentation for the custom spatial transcriptome–metabolome integration analyses used in the manuscript:

**Spatial multi-omic atlas profiles early organogenesis of a Carnegie stage 12 human embryo**

The repository focuses on the custom components of the analysis workflow, including JSRS-related coordinate integration, metabolite resampling, registration quality control, interpolation robustness analysis and external reference benchmarking. It also includes small example files to demonstrate input formats and code execution.

Raw human embryo imaging files, large expression matrices and complete intermediate objects are not included in this repository. Standard analyses performed with established software packages, such as Seurat-based preprocessing and clustering, are described in the manuscript Methods and summarized in the documentation.

- `outputs/` is the default output location for example runs.

## What is included

### 1. Initial coordinate alignment

The manuscript used the transcriptomic coordinate system as the reference coordinate system for cross-modal integration. The full registration workflow is described in `docs/registration_workflow.md`.

The script below demonstrates an initial rigid alignment step using point-cloud rasterization and SciPy/Powell optimization:

```bash
python code/run_initial_rigid_registration.py \
  --source-coords example_data/msi_coords_before_cs12_3.csv \
  --target-coords example_data/rna_coords_cs12_3.csv \
  --output outputs/initial_aligned.csv
```

This script estimates approximate rotation and translation parameters. In the manuscript workflow, this initial alignment was followed by SURF-based feature matching and image-based refinement before downstream resampling.

### 2. Area-overlap weighted metabolite resampling

The primary JSRS resampling strategy maps registered MSI features onto transcriptomic spatial bins using overlap-area weighted averaging. The implementation is provided in:

```text
code/used_upsample_analysis_Module.py
code/run_area_weighted_resampling.py
```

Example command:

```bash
python code/run_area_weighted_resampling.py
```

The example input paths and parameters are defined in `code/run_area_weighted_resampling.py`, which calls the original module `code/used_upsample_analysis_Module.py`. The output table contains RNA-bin coordinates and resampled metabolite intensities.

### 3. Alternative resampling methods

Two alternative resampling strategies are included for robustness assessment:

- nearest-neighbour resampling;
- inverse-distance weighted resampling.

The implementation is provided in:

```text
code/jsrs/alternative_resampling.py
code/run_nearest_idw_resampling.py
```

Example command:

```bash
python code/run_nearest_idw_resampling.py \
  --rna-coords example_data/rna_coords_cs12_3.csv \
  --msi-coords example_data/msi_coords_aligned_cs12_3.csv \
  --msi-matrix example_data/metabolite_matrix_cs12_3.csv \
  --output-nearest outputs/nearest_resampled.csv \
  --output-idw outputs/idw_resampled.csv
```

These alternative strategies were used to test whether downstream metabolomic conclusions were robust to the choice of resampling method.

### 4. Interpolation robustness analysis

The folder `analysis/interpolation_robustness/` contains R scripts for comparing metabolite matrices generated by different resampling methods.

The analyses include:

- cluster-level metabolite abundance correlation;
- metabolite-based cluster similarity comparison;
- all cluster-pair log2FC concordance;
- top-feature overlap;
- log2FC direction agreement.

These scripts start from cluster-level metabolite summary tables generated from resampled matrices and produce robustness summary tables and plots.

### 5. Registration quality control

The folder `analysis/registration_qc/` contains scripts for registration quality assessment.

The QC workflow includes:

- preparing coordinate masks;
- generating before/after registration overlay plots;
- calculating anatomical landmark errors;
- calculating tissue-mask overlap.

An example landmark table is provided:

```text
analysis/registration_qc/example_landmarks.tsv
```

The landmark table uses pre-defined anatomical landmarks and is intended to demonstrate the expected input format for the QC scripts.

## Input files

The example data files illustrate the required input formats.

### Transcriptomic coordinates

`example_data/rna_coords_cs12_3.csv`

Required columns:

```text
x, y
```

Optional columns:

```text
bin_id, barcode, _index
```

If no bin identifier is provided, the scripts generate one automatically.

### Registered MSI coordinates

`example_data/msi_coords_aligned_cs12_3.csv`

Required columns:

```text
x, y
```

Optional columns:

```text
spot_id, Spot_index, barcode
```

### MSI metabolite matrix

`example_data/metabolite_matrix_cs12_3.csv`

Expected format:

```text
MSI spot × metabolite feature
```

The first column should contain MSI spot identifiers or the row names should correspond to MSI spot identifiers. If the matrix orientation is transposed, the code attempts to detect and correct the orientation when possible.

### Cluster metadata

`example_data/rna_bin_cluster_metadata.tsv`

Required columns for cluster-level summaries:

```text
bin_id, section_id, cluster
```

## Installation

### Python environment

The Python scripts were written for Python 3.9 or later.

Required packages:

```text
numpy
pandas
scipy
scikit-image
matplotlib
```

Install with pip:

```bash
pip install -r requirements.txt
```

Alternatively, create a conda environment:

```bash
conda env create -f environment.yml
conda activate jsrs-spatial-omics
```

The R scripts require commonly used analysis packages including:

## Notes on interpretation

The resampling scripts operate on registered coordinates and processed MSI intensity matrices. They do not perform raw MSI peak picking, raw image preprocessing or complete spatial transcriptomic preprocessing.

The registration QC scripts evaluate the quality of cross-modal coordinate alignment using anatomical landmarks and tissue-mask overlap. They do not automatically define anatomical landmarks.

The metabolite resampling outputs represent spatially resampled relative MSI intensities and should be interpreted as regional metabolite abundance, not as direct measurements of metabolic flux.
